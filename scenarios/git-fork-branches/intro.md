# Git — Fork, branches & workflow collaboratif

Dans ce TP vous allez reproduire le workflow Git utilisé quotidiennement en équipe :
cloner un dépôt existant, créer des branches, maîtriser le staging et pousser votre travail.

## Contexte

> **Sur GitHub**, le premier geste est de **forker** un dépôt (en faire sa propre copie)
> avant de le cloner localement. Ici, le dépôt distant est simulé en local pour
> s'affranchir de l'authentification — les commandes Git sont identiques.

## Ce que vous allez pratiquer

| Étape | Compétence |
|-------|-----------|
| 1 | Cloner un dépôt et explorer ses branches locales et distantes |
| 2 | Créer une branche de feature, créer des fichiers, commiter |
| 3 | Maîtriser `git add`, `git commit -am` et `.gitignore` |
| 4 | Lire l'historique, créer un alias `git lg` et pousser sa branche |

## Pré-requis

- Un terminal Linux (fourni par Killercoda)
- L'environnement sera initialisé à la première étape via un script de setup

Passez à l'**Étape 1** pour démarrer.
