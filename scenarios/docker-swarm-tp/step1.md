# Étape 1 — Initialiser le Swarm

## Objectifs

1. Initialisez le Swarm sur cette machine (elle devient le **manager**)
2. Vérifiez l'état du Swarm avec `docker info` — repérez la ligne concernant le mode Swarm
3. Listez les nœuds du cluster — observez le rôle et la disponibilité du nœud manager

> 💡 Sur un vrai cluster, vous récupéreriez la commande `docker swarm join` affichée à
> l'initialisation pour y ajouter des workers. Vous pouvez aussi retrouver ce token
> à tout moment avec une commande dédiée (`docker swarm join-token`).

> 💡 Comment promouvoir un worker en manager ? Quelle est la différence entre un manager
> **Leader** et un manager classique dans un cluster multi-managers ?

Cliquez sur **Check** quand c'est fait.
